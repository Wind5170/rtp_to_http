# udpxy_health_check.py UDPXY 服务器健康检查工具

## 功能概述

用于测试 udpxy 服务器的可用性和速度，通过访问组播地址验证服务器是否正常工作。支持多种输入格式，生成多个输出文件。

## 处理流程

### 1. 文件选择

```
弹出文件选择对话框（默认目录: udpxy_servers\collected）
    ↓
选择多个 udpxy 服务器地址文件
    ↓
验证文件并解析服务器列表
```

### 2. 地区和运营商识别

```
从文件名或目录名提取地区和运营商
    ↓
从文件内容解析服务器列表，支持多种格式
```

### 3. 测试配置

```
命令行参数：
├── -p, --protocol: 测试协议（rtp 或 udp，默认 rtp
├── -f, --file: 指定服务器文件路径
├── -w, --workers: 并发测试线程数（默认 30
├── -a, --all: 强制测试所有地址（包括标记为无效的
└── -t, --test-invalid: 是否测试标记为无效的服务器

读取 test_udpxy_config.txt 配置文件，获取各地区的测试组播地址
```

### 4. 测试执行

```
并发测试（ThreadPoolExecutor）
    ↓
对每个服务器调用 test_udpxy_server(server, test_url)
    ↓
├─ 下载≥16KB ──→ [√] 有效，记录耗时
└─ 下载<16KB ──→ [×] 无效
```

### 5. 结果保存

```
生成多个输出文件：
├─ 1. {源文件名}_health_checked.txt: 单个文件的测试结果（带有效/无效标记）
├─ 2. udpxy_servers_active.txt: 有效服务器汇总（追加模式）
├─ 3. nodes.txt: udpxy_servers_active.txt 的副本（供 build_udpxy_playlist.py 使用）
└─ 4. udpxy_servers_history.txt: 测试历史记录（追加模式）
```

## 变量说明

| 变量 | 说明 |
|------|------|
| `regions` | {地区: {active: [], failed: []}} |
| `active` | 当前有效的服务器列表 |
| `failed` | 之前测试失败的服务器列表 |
| `active_invalid` | 本次测试失败的活动服务器 |
| `valid_failed` | 本次测试有效的失败服务器（恢复） |
| `invalid_failed` | 本次测试仍然失败的服务器 |
| `server_speed_map` | 服务器: 耗时 映射，用于排序 |

## 使用方法

```bash
py udpxy_health_check.py
```

### 命令行参数

```bash
# 基本使用
py udpxy_health_check.py

# 指定测试协议（rtp 或 udp）
py udpxy_health_check.py -p udp

# 指定服务器文件
py udpxy_health_check.py -f udpxy_servers/collected/江苏电信.txt

# 指定并发数
py udpxy_health_check.py -w 50

# 强制测试所有服务器（包括标记为无效的）
py udpxy_health_check.py -a

# 测试标记为无效的服务器
py udpxy_health_check.py -t
```

## 测试逻辑

`test_udpxy_server(server, test_url)` 函数：

1. 构建测试 URL（移除重复的 http:// 前缀）
2. 使用 requests 访问，设置超时（3 秒连接超时，5 秒读取超时）
3. 检查是否下载了至少 16KB 数据
4. 返回：(server, 是否有效, 耗时)

## 注意事项

1. **编码问题**：使用 utf-8 编码读取和写入文件
2. **地区识别**：优先从文件名识别，其次从目录名识别
3. **去重逻辑**：active 文件采用追加模式，按服务器地址去重
4. **服务器排序**：测试完成后，有效服务器按耗时从小到大排序
5. **协议选择**：支持 rtp 和 udp 两种协议进行测试
6. **并发控制**：并发数不超过活动服务器数量，避免资源浪费

## 运营商识别

支持自动识别以下运营商：
- 电信（默认）
- 移动
- 联通

## 测试前检查

程序会先检查每个地区是否配置了测试地址：
- 有测试地址：正常测试该地区的服务器
- 无测试地址：跳过该地区，显示警告信息

## 输出统计

```

测试完成: 有效 50, 无效 10, 总计 60

```
