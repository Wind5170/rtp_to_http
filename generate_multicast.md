# generate_multicast.py 组播文件生成工具

## 功能概述

根据 channel_list.txt 文件生成标准化的组播地址文件（multicast+日期.txt）。主要用于将频道数据列表转换为可被 multicast_health_check.py 或 build_udpxy_playlist.py 使用的组播格式文件。

## 处理流程

```
读取 source/channel_list.txt 文件
    ↓
跳过第一行说明行（分类,频道名称,分辨率,状态,组播地址）
    ↓
逐行解析数据（每行5列）
    ↓
跳过状态不为空的行（已标记为无效的频道）
    ↓
按分类分组，保持分类出现顺序
    ↓
生成输出文件到 multicast_generate/ 目录
    ↓
添加更新时间分类和时间戳
```

## 文件格式

### 输入文件格式（source/channel_list.txt）

```
分类,频道名称,分辨率,状态,组播地址
央视,CCTV1,SD,,239.49.0.1:8000
央视,CCTV2,HD,,239.49.0.2:8000
央视,CCTV4K,4K,,239.49.0.3:8000
卫视,湖南卫视,SD,,239.49.0.10:8000
卫视,浙江卫视,HD,,239.49.0.11:8000
```

#### 列说明

| 列 | 说明 | 示例 |
|----|------|------|
| 第1列 | 分类 | 央视、卫视、央视频道 |
| 第2列 | 频道名称 | CCTV1、湖南卫视 |
| 第3列 | 分辨率 | SD、HD、4K（可为空） |
| 第4列 | 状态 | 空表示有效，非空表示无效 |
| 第5列 | 组播地址 | 239.49.0.1:8000 |

### 输出文件格式（multicast_generate/multicast_YYYYMMDD_HHMMSS.txt）

```
央视,#genre#
CCTV1,rtp://239.49.0.1:8000
CCTV2-HD,rtp://239.49.0.2:8000
CCTV4K-4K,rtp://239.49.0.3:8000

卫视,#genre#
湖南卫视,rtp://239.49.0.10:8000
浙江卫视-HD,rtp://239.49.0.11:8000

更新时间,#genre#
2026/04/30 12:00,https://taoiptv.com/time.mp4
```

## 处理规则

### 1. 频道名称处理

| 分辨率 | 显示名称 | 示例 |
|--------|---------|------|
| SD 或空 | 仅频道名 | `CCTV1` |
| HD | 频道名-HD | `CCTV2-HD` |
| 4K | 频道名-4K | `CCTV4K-4K` |
| 其他 | 频道名-分辨率 | `频道名-分辨率` |

### 2. 组播地址处理

| 输入格式 | 输出格式 | 示例 |
|---------|---------|------|
| `239.49.0.1:8000` | 自动添加 rtp:// | `rtp://239.49.0.1:8000` |
| `rtp://239.49.0.1:8000` | 保持不变 | `rtp://239.49.0.1:8000` |
| `udp://239.49.0.1:8000` | 保持不变 | `udp://239.49.0.1:8000` |

### 3. 分类处理

- 按输入文件中的分类出现顺序输出
- 每个分类下按频道出现顺序输出
- 相同分类的频道连续输出
- 分类之间用空行分隔

### 4. 状态过滤

- 状态为空的行：正常处理并输出
- 状态非空的行：跳过（视为无效频道）

### 5. 更新时间

- 在文件末尾添加"更新时间"分类
- 格式：`更新时间,#genre#`
- 时间戳格式：`YYYY/MM/DD HH:MM`

## 数据处理示例

### 输入数据

```
央视频道,CCTV1,SD,,239.49.0.1:8000
央视频道,CCTV2,HD,,239.49.0.2:8000
央视频道,CCTV4K,4K,,239.49.0.3:8000
央视频道,CCTV6,SD,无效,239.49.0.4:8000
卫视,湖南卫视,SD,,239.49.0.10:8000
卫视,浙江卫视,HD,,239.49.0.11:8000
```

### 处理过程

1. 第1行是说明行 → 跳过
2. 第2行状态为空 → 处理，输出 `CCTV1,rtp://239.49.0.1:8000`
3. 第3行状态为空 → 处理，输出 `CCTV2-HD,rtp://239.49.0.2:8000`
4. 第4行状态为空 → 处理，输出 `CCTV4K-4K,rtp://239.49.0.3:8000`
5. 第5行状态为"无效" → 跳过
6. 第6行状态为空 → 处理，输出 `湖南卫视,rtp://239.49.0.10:8000`
7. 第7行状态为空 → 处理，输出 `浙江卫视-HD,rtp://239.49.0.11:8000`

### 输出结果

```
央视频道,#genre#
CCTV1,rtp://239.49.0.1:8000
CCTV2-HD,rtp://239.49.0.2:8000
CCTV4K-4K,rtp://239.49.0.3:8000

卫视,#genre#
湖南卫视,rtp://239.49.0.10:8000
浙江卫视-HD,rtp://239.49.0.11:8000

更新时间,#genre#
2026/04/30 12:00,https://taoiptv.com/time.mp4
```

## 注意事项

1. **编码问题**：使用 utf-8 编码读取和写入文件
2. **说明行**：第一行如果包含"分类,频道名称,分辨率,状态,组播地址"会被自动跳过
3. **状态过滤**：状态非空的行视为无效，不会输出到结果中
4. **分类顺序**：输出按输入文件中分类的首次出现顺序排列
5. **时间戳**：每次运行生成的文件名包含不同的时间戳
6. **自动创建**：如果输出目录不存在，会自动创建

## 与其他工具的配合

1. **multicast_health_check.py**：使用生成的组播文件进行健康检查
2. **build_udpxy_playlist.py**：将生成的组播文件复制到 source/multicast/ 目录作为模板
3. **extract_udpxy_multicast.py**：从各种格式提取组播地址后，可用此工具生成标准化文件

## 错误处理

| 错误情况 | 处理方式 |
|---------|---------|
| 输入文件不存在 | 打印错误信息并退出 |
| 文件格式错误（列数不为5） | 跳过该行并打印警告 |
| 组播地址为空 | 跳过该行并打印警告 |
| 写入文件失败 | 打印错误信息并退出 |
